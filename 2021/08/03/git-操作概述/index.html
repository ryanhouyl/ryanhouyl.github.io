是什么

git 是由 Linus 开发的分布式版本控制系统，分布式的操作和对于分支的管理甩开SVN等其他版本管理工具几条街；而GitHub 免费为 git 提供存储空间，两者相互结合为程序员管理代码和开源学习带来了极大的便利

常看命令帮助

git <命令名> --help查看命令的详细操作，非常实用，如：

git commit --help

克隆远程版本库

不管是学习别人的代码，还是把自己远程的代码同步下来，克隆命令都是必须的，操作如下：

git clone 远程版本库地址 [克隆到本地的文件名，也可以不写]
如：

git clone https://github.com/Ryan-hou/node xxx

在本地创建 git 仓库

可以使用 GitHub Desktop 软件创建；也可以使用 IDEA create Git Repository 创建；或者直接使用命令行创建，使用命令行创建的截图如下：

在本地仓库添加和提交文件

  • 先使用 git status 命令查看当前工作分支下的文件状态：是否提交到暂存区？是否 commit ？是否提交后又改动？保持工作区 clean 是好习惯

  • 如有变动，可以使用 git diff 查看改动了什么内容，确认无误后，提交到本地仓库

  • 提交文件
    git add <file> // 把工作区文件添加到暂存区
    git commit -m '本次提交的注释'

实际应用

  • 如果仅仅是为了在本地管理自己的 demo 代码，就直接在本地创建相应的 Git 仓库即可

  • 如果需要在 GitHub 上托管自己的代码，可以直接用 Idea 发布到 GitHub 进行代码管理

  • 更常见的方式是，把 GitHub 上的代码 fork 到本地，然后在本地的 Git 仓库进行代码管理，修改完毕后，再同步到 GitHub

  • 配置 gitignore 文件，管理需要进行同步的代码

以上就是使用 git 和 GitHub 管理自己代码的基本操作流程，只是提供了一个主要框架，还有很多细节和更多的命令需要进一步学习
