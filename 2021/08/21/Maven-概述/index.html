Maven 是什么

基于 POM(Project Object Model) 的Java项目构建工具。那什么是项目构建工具？

构建工具是将软件项目构建相关的过程自动化的工具，构建一个软件项目一般包含以下多个过程：生成源码，从源码生成项目文档，编译源码，打包，安装，发布等

Maven 哲学

事实上 Maven 开发者并不仅仅把 Maven 作为一个项目构建工具，而是有自己 Philosophy

Ant is simply a toolbox whereas Maven is about the application of patterns in order to achieve an infrastructure which displays the characteristics of visibility, reusability, maintainability, and comprehensibility

通过 a set of patterns 来实现统一的 project management

Maven 的核心概念

Maven 的核心是 POM 文件，POM 文件描述了项目的资源，如源码，测试代码，依赖等，POM 文件位于项目的根目录下

pom.xml 文件

当你执行一条Maven命令的时候，你会传入一个pom文件。Maven会在该pom文件描述的资源上执行该命令

POM 文件描述的是构建什么，而不是如何构建。如何构建取决于 Maven 的构建阶段和目标，当然，如果有必要，可以向Maven构建阶段添加自定义的目标

生命周期，阶段与目标 – life cycles, phases, goals

Maven的构建过程被分解为构建生命周期、阶段和目标

一个构建周期由一系列的构建阶段组成，每一个构建阶段由一系列的目标组成。当你运行Maven的时候，你会传入一条命令。这条命令就是构建生命周期、阶段或目标的名字。如果执行一个生命周期，该生命周期内的所有构建阶段都会被执行。如果执行一个构建阶段，在预定义的构建阶段中，所有处于当前构建阶段之前的阶段也都会被执行

依赖 – dependencies

  • 项目依赖

    Maven执行时，其中一个首要目标就是检查项目的依赖。依赖是你的项目用到的jar文件（java库）。如果在本地仓库中不存在该依赖，则Maven会从中央仓库下载并放到本地仓库。本地仓库只是你电脑硬盘上的一个目录。你可以根据需要制定本地仓库的位置。你也可以指定下载依赖的远程仓库的地址。

  • 外部依赖

    Maven 的外部依赖指的是不在 Maven 仓库中的依赖，他可能位于你本地硬盘的某个地方，比如 web 应用的 lib 目录下，这里的外部是针对Maven仓库系统而言的，举个例子：

  • 快照依赖

    快照依赖是指那些还在开发中的依赖，与其经常的更新版本号来获取最新版本，不如直接依赖项目的快照版本。总是下载快照依赖可以确保本地仓库中的每一个 build 版本都是最新的

Maven仓库

Maven 仓库就是存储 jar 包和一些元数据信息的目录，其中的元数据即 pom 文件，描述了该 jar 包属于哪个项目，以及 jar 需要的外部依赖，该元数据信息使得Maven可以递归的下载所有的依赖，直到整个依赖树都下载完毕放到本地仓库中

插件 – Build plugins

构建插件可以向构建阶段中增加额外的构建目标。如果Maven标准的构建阶段和目标无法满足项目构建的需求，你可以在POM文件里增加插件。Maven有一些标准的插件供选用，如果需要你可以自己实现插件。

Maven 目录结构

Maven 有一个标准的目录结构，如果你在项目中遵循 Maven 的目录结构，就无需在 pom 文件中指定源代码，测试代码等目录

Maven 构建配置

Maven 构建配置使你能使用不同的配置来构建项目，不用创建两个独立的 pom 文件，你只需使用不同的构建配置指定不同的配置文件，然后使用该配置文件构建项目即可。profile 属性中的值将会覆盖其上层的，位于 pom 文件中的配置; 构建配置可以在 setting.xml 文件中指定，也可以使用 Maven 命令；举个例子：

Maven 的继承

所有的 Maven pom 文件都继承自一个父 pom，如果没有指定父 pom，该pom 文件继承自根 pom；可以让一个 pom 文件显示地继承另一个 pom 文件，这样，可以通过修改公共的 pom 文件的设置来修改所有的子 pom 文件的设置，在 pom 文件的起始处指定父 pom；子 pom 文件可以覆盖父 pom 文件的设置；使用 mvn help:effective-pom 命令可以得到有效的 pom

运行 Maven

当你安装好了 Maven，并且在项目的根目录下创建了 POM 文件，就可以在项目上运行 Maven 了。运行 Maven 只需要在命令行执行 "mvn" 命令即可，当执行 “mvn” 命令时，将构建周期，阶段或者目标作为参数传进去，Maven 就会执行他们。比如：mvn clean install
