是什么

用于定义 XML 文档的合法构建模块，我们需要根据 XML 约束文档的定义来编写合法的 XML 实例文档

为什么要学习 XML 约束

  • 编写框架时根据需要的 XML 文件来定义相应的 XML 约束文档

DTD 实体定义

在 DTD 中可以定义和使用 ENTITY(Schema 约束不支持)，用于为一段内容创建一个别名，以后在 XML 文档或者该DTD 文档中就可以使用别名来引用这段内容了，主要的目的就是为了复用，分为引用实体和参数实体

引用实体：用于 XML 文件

参数实体：用于 DTD 文件本身

Schema 约束

是什么

XML Schema 也是一种用于定义和描述 XML 文档结构与内容的模式语言，其出现是为了克服 DTD 约束的局限性

对比 DTD

Schema 作为 DTD 约束的继任者，相比 DTD 约束有以下优势：

  • XML Schema 符合 XML 语法结构，方便我们使用 DOM 等解析，也方便使用 XSLT 进行转换
  • 支持数据类型，而且可以自定义数据类型
  • Schema 定义约束的能力更加强大，可以对 XML 实例文档作出细致的语义限制
  • 支持名称空间

如何使用

  • 定义 xsd 文件：定义名称空间和各种元素、属性即其对应的约束
  • 使用 xsd 文件：在 XML 文件中引入定义的 xsd 文件，按约束来书写元素、属性

定义简单 XML 元素(标签)

定义简单元素：简单元素指那些仅包含文本的元素，不包含任何其他元素或者属性

语法：
<xs:element name="xxx" type="yyy" />
xxx 指元素的名称，yyy 指元素的数据类型，XML Schema 拥有很多的内建数据类型，常用的有：
xs:string, xs:integer, xs:boolean, xs:date, xs:decimal 等

还可以定义元素的默认值与固定值：

定义 XML 属性

简单元素无法拥有属性，假如某个元素拥有属性，那么它就是某种复合类型。但是属性本身总是作为简单类型被声明的，其语法类似于简单元素的定义

定义属性的语法：
<xs:attribute name="xxx" type="yyy" />

name 和 type 的使用同上面简单元素的语法，属性也可以指定默认值和固定值，而且可以指定可选的和必须的属性(在默认情况下，属性是可选的，如需规定属性为必选，需要使用 use)：

<xs:attribute name="lang" type="xs:string" use="required" />
