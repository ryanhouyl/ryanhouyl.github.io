为什么要学习 Template 的设置

简单来说一句话：提高开发效率；本着所有机械重复的工作都应该自动化处理的原则，我们有必要学习一下设置 IDEA 的 Live Templates 和 File and Code Templates

需要的准备

  • 基本的 Apache Velocity语法
  • 开发工具 IntelliJ IDEA
  • 本文介绍主要的操作流程和重要的使用，具体详细的内容一定要记得去看官方文档：IDEA Help 和 Apeche Velocity

使用和配置 Live Templates

是什么

Live templates let you insert frequently-used or custom code constructs into your source code file quickly, efficiently, and accurately. They contain the predefined code fragments.

举个例子：在 IDEA 的 Java 代码文件中想要快速创建一个 main 方法，可以使用 IDEA 提前定义好的 Live Tempate，输入 psvm 后，激活该 Template，就会自动把 psvm 转为 main 方法的 Java 代码：public static void main(String[] args){}

记不住 Template 的简写字母也没有关系，可以使用 Command + J查看当前所有可用的 Live Templates，选中想要的 Template 后，还可以查看当前选中的 Template 的具体定义，如下图所示：

找到配置的位置：先进入 Preferences，选择Editor中的Live Templates菜单就可以进入自定义配置了，示意图如下，其中标红部分就是下面重点要介绍的点：

  • 最关键的地方就是 Template Text 部分和 Edit variables部分了，这两个部分用来定义具体的模板内容，在 Template Text 中可以使用固定的文本，也可以使用变量$<variable_name>$，其中$END$变量和$SELECTION$是内置的变量，当创建除了这两个变量以外的变量，就可以使用Edit variables按钮来编辑该变量了

使用和配置 File and Code Templates

是什么

File templates are specifications of the contents to be generated when creating a new file.They let you create the source files that already contain some initial code.

举个例子，当我们使用 IDEA 新建一个 Java 类文件时，IDEA 会自动给我们生成导入包的语句和类名等信息，这些就是通过定义 Template 来实现的，模板的定义如下图:

定义和修改 Template

找到配置的位置：先进入 Preferences，选择Editor中的File and Code Templates菜单就可以进入自定义配置了，示意图如下，其中标红部分就是下面重点要介绍的点：

上图中的 Files 栏就是已经定义好的模板，比如我们常用的 Class，Interface等，我们可以选中然后在右侧窗口修改，如何修改就是重点了，在右侧窗口下面的 Description 中其实已经有具体的描述了，模板内容可以定义的内容有：

  • Fixed text: 固定文本，也就是一些字符串常量
  • File template variables：分为 Predefined template variables，如${NAME}，${PACKAGE_NAME}等；Custom template variables，自定义的模板变量可以使用 VTL 指令来赋值，如#set( $MyName = "John Smith" )
  • #parsed directive：用来引入在 Includes Tab标签栏中定义的模板文件，如：#parse("File Header.java")
  • Other VTL constructs：使用其他的一些 VTL 语法，比如#[[]]#，## 等

接下来就可以根据上面的定义，来根据自己的实际编写模板文件了，比如我们可以修改 get/set 方法，IDEA 中默认的模板文件是不可以修改的，当然我们一般也不会修改系统自带的模板，而是新建一个本地的模板来用，在新建的模板中使用 VTL 语法来定义自己的模板文件，举例如下图：
