Map 概述

定义

Map 是用于保存具有映射关系的数据，即保存键值对的集合

使用规范

  • Map 中的 key 值唯一，即同一个 Map 对象的任何两个 key 值通过 equals 方法将返回 false，key 值是 Map 集合中数据的唯一标识
  • key 与 value 存在单向一对一关系，即通过指定的 key 总能找到唯一确定的 value

Map 典型实现之HashMap

需要知道的几个点

1.同 HashSet 一样(HashSet 实际上是对 HashMap 的简化，保留了 HashMap 的 key 值，HashSet 中的各种方法实际上是在调用 HashMap 的方法)，HashMap 不能保证其中 key-value 键值对的顺序，而且不能保证键值对的顺序不发生变化(由 HashMap 的自动扩容引起，后面再展开)

2.HashMap 采用的数据结构为：数组 ＋ 链表：HashMap 底层就是一个数组结构，数组中的每一项又是一个链表。当新建一个 HashMap 时，就会初始化一个数组，数组的元素为 Entry 键值对对象，它持有指向下一个元素的引用，这样就构成了链表

  • 往 HashMap 中 put 元素时，先根据 key 的 hashCode 重新计算 hash 值，根据 hash 值得到这个元素在数组中的位置，如果数组的该位置存在其他元素，那么在这个位置的元素将以链表的形式存放，新加入的元素放在链头；如果数组该位置没有元素，则直接放到该位置即可

  • 当需要取出一个 Entry 对象时，也是先由 hash 算法找到其在数组中的存储位置，再根据 equals 方法从该位置的链表中取出要找的 Entry

4.HashMap 中的 hash 算法

  • hash(int h) 方法根据 key 的hashCode 重新计算一次散列，对于任意给定的对象，只要他的 hashCode() 返回值相同，那么程序调用 hash(int h) 方法所计算得到的 hash 值总是相同的

  • 任何 hash 算法都需要考虑hash 冲突的问题，为了降低 hash 冲突的概率，需要选择好的hash算法，使得hash值的分布尽可能随机均匀

5.HashMap 的扩容问题

  • 数组长度固定时，当 HashMap 中的元素越来越多时， hash 冲突的几率会越来越高，为了提高查询效率，就要对 HashMap 的数组进行扩容，扩容后原数组的数据必须重新计算在新数组中的位置，并放进去，因此相当耗费性能(这也解释了 HashMap 中键值对的顺序不能保证不变化)

  • 扩容发生的条件为：当 HashMap 中元素的个数超过数组大小的装填因子 loadFactor时进行扩容。在默认情况下，数组大小为16， loadFactor 大小为 0.75，那么当 HashMap 中元素的个数超过 16*0.75 = 12 时，就把数组大小扩大一倍，变为 32

  • 关于装填因子的大小也需要进行适当的权衡，这也是我们经常遇到的问题，时间与空间的权衡问题。对于使用链表法散列表来说，查找一个元素的平均时间是 O(1 + a)，装填因子越大，空间的利用越充分，但是查找效率下降；装填因子变小，查找效率提高，但是空间利用率下降
